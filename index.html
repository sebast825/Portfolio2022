Stripe Subscription Integration

The reliable and scalable system for managing recurring payments on SaaS platforms

The Challenge

Building a standalone payment system from scratch required handling Stripe's asynchronous webhooks, which can arrive out of order or be retransmitted multiple times. A self-contained solution was needed that guaranteed idempotency and consistency without relying on existing infrastructure.

The Solution

I designed and developed a full-stack system with a .NET 8 backend and React frontend, implementing idempotent webhook logic and timestamp comparison to maintain consistency despite disordered events.

Technical Highlights

  • Idempotent Webhook Processing: Each Stripe event is logged and validated to prevent duplicates from automatic retries.
  • Timestamp-based Consistency: Events are compared by creation time to avoid outdated webhooks overwriting current state.
  • Race Condition Resolution: Special handling for initial payments where payment confirmation may arrive before subscription creation.
  • Request Queue Pattern: Frontend interceptor prevents multiple simultaneous token refresh attempts.

Achieved Impact

  • Zero duplications thanks to processed event logging and idempotency validation.
  • Robust race condition handling for edge cases in payment flow sequencing.
  • Modular architecture separating checkout, webhooks, and state management, enabling scaling to thousands of daily transactions.

Secure Authentication & Authorization API

A production-ready, scalable authentication service with advanced security features

The Challenge

Modern applications require robust security against evolving threats like DDoS, credential stuffing, and brute force attacks. A standalone authentication service was needed that could handle high traffic while maintaining strict security protocols and comprehensive audit trails.

The Solution

I architected and developed a complete authentication API using Clean Architecture with .NET 8, implementing JWT with refresh tokens, IP-based rate limiting, and a queue-based system (RabbitMQ) to prevent database saturation during attack scenarios.

Key Features

  • Queue-Based Attack Mitigation: RabbitMQ integration to offload failed login attempts during brute force attacks.
  • Comprehensive Audit Trail: Detailed logging of all authentication events with IP, device, and timestamp data.
  • JWT with Refresh Tokens: Secure token management with automatic invalidation on logout.
  • Defense-in-Depth: Multiple security layers including rate limiting, and attempt logging.
  • Database Protection: Queue persistence prevents connection pool exhaustion during attacks.
